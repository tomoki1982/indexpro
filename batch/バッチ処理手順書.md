# バッチ処理手順書

このフォルダは、個人PCで更新担当者が使うファイルだけを置いています。

## 使うファイル

- `run_ui.bat`
  - 個人PCでローカル画面を開く
- `push_to_github.bat`
  - GitHub にコードを反映する
- `open_actions_page.bat`
  - GitHub Actions 画面を開く

## 更新の流れ

更新作業は、毎回この順番です。

### 1. 個人PCでデータを更新する

Googleスプレッドシートなど、公開版が読んでいる元データを更新します。

### 2. 必要ならローカル画面で確認する

`run_ui.bat` を実行して、個人PCで動作確認します。

確認する内容:
- メーカー検索
- 代理店・取扱店検索
- 推定代理店検索
- 推定競合検索

### 3. コード修正がある場合だけ GitHub に反映する

画面やロジックも変更した場合だけ `push_to_github.bat` を実行します。

このバッチで行うこと:
- `git add .`
- `git commit`
- `git push`

データだけ更新した場合は、この手順は不要です。

### 4. GitHub Actions を手動実行する

`open_actions_page.bat` を実行して、GitHub の `Actions` 画面を開きます。

そのあと `Manual Redeploy Streamlit` を実行します。

役割:
- Streamlit 公開版を再デプロイする
- 最新データを読み直させる

### 5. 公開版を確認する

公開URLを開いて、反映を確認します。

公開URL:
- `https://indexpro-search.streamlit.app/`

確認する内容:
- 画面が開く
- 検索が動く
- 画面下の最終更新日が想定どおり

## 一番シンプルな覚え方

- データ更新
- 必要ならローカル確認
- 必要なら GitHub に push
- GitHub Actions 実行
- 公開版確認

## 補足

- 一般利用者はこのフォルダを使いません。
- 更新担当者だけが使う前提です。
